# 信息化项目验收文档编写技能

## 概述

本技能专为信息化工程项目验收阶段的文档编写而设计，提供标准化的文档模板和编写指导。支持多种项目文档类型，包括标准验收文档和先建后批场景下的补充文档。

## 核心功能

### 标准验收文档
- **项目概要设计** - 项目总体设计说明
- **项目详细设计** - 模块级详细设计说明
- **项目产品手册** - 用户操作指南
- **测试报告** - 系统测试结果总结
- **项目架构设计报告** - 技术架构详细说明
- **项目数据库设计报告** - 数据库设计详细说明
- **项目运维报告** - 运维体系说明

### 先建后批补充文档
- **项目可研报告** - 已建设项目补充可行性研究
- **项目立项报告** - 已建设项目补充立项申请

## 使用方法

### 1. 使用参考模板手动编写

直接参考 `references/` 目录下的模板文件，根据项目实际情况填写内容。

**示例：编写项目概要设计**
```bash
# 打开模板文件
open references/概要设计模板.md

# 复制内容到新文档
cp references/概要设计模板.md 你的项目_概要设计.md

# 根据项目实际情况修改内容
```

### 2. 使用模板生成脚本批量生成

运行模板生成脚本，根据项目信息自动生成文档框架。

```bash
python3 scripts/generate_template.py
```

生成的文档将保存在 `generated_docs/` 目录下。

### 3. 验证文档质量

使用验证脚本检查文档的完整性和规范性。

```bash
# 简化验证（推荐）
python3 scripts/simple_validate.py

# 详细验证
python3 scripts/validate_document.py
```

### 4. 检查文档一致性
使用一致性检查脚本检查多个文档之间的一致性。
```bash
python3 scripts/check_consistency.py
```

### 5. 转换为Word文档
使用markdown转Word脚本将文档转换为Word格式，便于提交和分享。
```bash
# 转换单个文件
python3 scripts/markdown_to_word.py -i references/概要设计模板.md -o output.docx -t "项目概要设计"

# 批量转换生成的文档
python3 scripts/markdown_to_word.py -d generated_docs/ -o word_output/
```

## 文档编写流程

### 标准验收文档编写流程

1. **准备阶段**：收集所有相关设计文件和技术文档
2. **选择模板**：根据文档类型选择对应模板
3. **框架填充**：先完成文档结构框架
4. **内容编写**：逐章节填充具体内容
5. **图表制作**：补充必要的架构图、流程图
6. **审核完善**：多轮审核，确保质量

### 先建后批补充文档编写流程

1. **现状梳理**：整理系统当前运行情况
2. **反向推导**：基于现状反推可研/立项要素
3. **效益分析**：基于实际运行数据进行效益分析
4. **风险评估**：识别当前系统存在的风险
5. **优化建议**：提出后续优化改进方向

## 文档质量要求

### 内容要求
- **完整性**：涵盖所有必要章节和内容点
- **准确性**：技术描述准确，数据真实可靠
- **一致性**：术语使用统一，格式规范
- **可读性**：逻辑清晰，语言简洁专业

### 格式要求
- **标题层级**：使用规范的标题层级（#、##、###）
- **图表使用**：重要设计点配架构图、流程图
- **代码规范**：技术文档中的代码示例要规范
- **引用标注**：引用外部资料需标注来源

### 审核要点
- **技术审核**：技术方案的可行性、先进性
- **业务审核**：业务需求的满足程度
- **规范审核**：是否符合行业标准和规范

## 脚本使用说明

### validate_document.py - 详细验证脚本
- 功能：对文档进行详细的格式和内容验证
- 使用：`python3 scripts/validate_document.py`
- 输出：详细的验证报告，包括格式问题和内容问题

### simple_validate.py - 简化验证脚本
- 功能：验证文档的基本结构完整性
- 使用：`python3 scripts/simple_validate.py`
- 输出：基本的验证结果，适合快速检查

### generate_template.py - 模板生成脚本
- 功能：根据项目信息批量生成文档模板
- 使用：`python3 scripts/generate_template.py`
- 输出：生成的文档保存在 `generated_docs/` 目录

### check_consistency.py - 一致性检查脚本
- 功能：检查多个文档之间的一致性
- 使用：`python3 scripts/check_consistency.py`
- 输出：一致性检查报告

## 使用示例

### 示例1：为新项目编写验收文档

```bash
# 1. 生成文档模板
python3 scripts/generate_template.py

# 2. 修改生成的文档
cd generated_docs
# 编辑 IT-2026-001_概要设计.md 等文件

# 3. 验证文档质量
cd ..
python3 scripts/simple_validate.py

# 4. 检查一致性（可选）
python3 scripts/check_consistency.py
```

### 示例2：为现有项目补充文档

```bash
# 1. 选择对应模板
cp references/可研报告模板.md 你的项目_可研报告.md

# 2. 根据实际情况填写
# 编辑 你的项目_可研报告.md

# 3. 验证文档
python3 scripts/simple_validate.py
```

## 注意事项

1. **保密要求**：项目文档可能包含敏感信息，注意保密
2. **版本管理**：文档应有明确的版本号和修订记录
3. **模板适配**：根据项目实际情况调整模板内容
4. **标准遵循**：参考国家相关标准和行业规范
5. **持续更新**：项目变更时及时更新相关文档

## 常见问题

### Q: 如何选择合适的文档类型？
A: 根据项目阶段和需求选择：
- 验收阶段：选择标准验收文档（概要设计、详细设计等）
- 先建后批：选择补充文档（可研报告、立项报告）

### Q: 模板中的占位符如何处理？
A: 模板中的 `[占位符]` 需要根据项目实际情况替换为具体内容

### Q: 如何确保文档质量？
A: 建议按照以下步骤：
1. 使用验证脚本检查基本结构
2. 请相关专家进行技术审核
3. 多轮修改完善
4. 用户验收确认

### Q: 文档编写遇到困难怎么办？
A: 可以：
1. 参考模板中的示例内容
2. 查阅相关技术文档
3. 咨询领域专家
4. 参考同类项目的文档
