# 快速使用指南

## 5分钟快速开始

### 第一步：生成文档模板

```bash
python3 scripts/generate_template.py
```

这将生成9个标准文档模板，保存在 `generated_docs/` 目录下。

### 第二步：修改文档内容

进入 `generated_docs/` 目录，根据项目实际情况修改以下文档：

1. **IT-2026-001_概要设计.md** - 项目总体设计
2. **IT-2026-001_详细设计.md** - 模块详细设计
3. **IT-2026-001_产品手册.md** - 用户操作指南
4. **IT-2026-001_测试报告.md** - 测试结果总结
5. **IT-2026-001_架构设计.md** - 技术架构说明
6. **IT-2026-001_数据库设计.md** - 数据库设计说明
7. **IT-2026-001_运维报告.md** - 运维体系说明
8. **IT-2026-001_可研报告.md** - 可行性研究（先建后批）
9. **IT-2026-001_立项报告.md** - 项目立项申请（先建后批）

### 第三步：验证文档质量

```bash
python3 scripts/simple_validate.py
```

确保所有文档都通过验证。

## 按需使用

### 场景1：只编写特定文档

如果只需要编写某个特定文档，可以直接使用对应的模板：

```bash
# 例如：只编写项目概要设计
cp references/概要设计模板.md 你的项目_概要设计.md
# 然后编辑 你的项目_概要设计.md
```

### 场景2：先建后批场景

如果需要为已建项目补充文档：

```bash
# 生成可研报告和立项报告
python3 scripts/generate_template.py
# 然后重点修改：
# - IT-2026-001_可研报告.md
# - IT-2026-001_立项报告.md
```

### 场景3：批量生成并验证并转换为Word

```bash
# 1. 生成所有文档
python3 scripts/generate_template.py

# 2. 验证文档质量
python3 scripts/simple_validate.py

# 3. 转换为Word格式（便于提交）
python3 scripts/markdown_to_word.py -d generated_docs/ -o word_output/
```

## 文档编写技巧

### 1. 填写文档信息
每个文档开头都有"文档信息"部分，需要填写：
- **文档编号**：项目编号-文档类型-版本号
- **文档名称**：项目名称 + 文档类型
- **编制单位**：你的公司/团队名称
- **日期**：实际的编制日期

### 2. 替换占位符
模板中的 `[占位符]` 需要替换为实际内容：
- `[项目名称]` → 实际项目名称
- `[模块名称]` → 实际模块名称
- `[功能描述]` → 实际功能描述

### 3. 补充具体内容
根据项目实际情况，补充：
- 项目背景和目标
- 技术架构和设计方案
- 功能模块和业务流程
- 测试结果和性能指标
- 运维方案和安全措施

### 4. 添加图表
在适当位置添加：
- 系统架构图
- 业务流程图
- 数据库ER图
- 网络拓扑图

## Word文档转换

### 为什么需要转换为Word？
- **便于提交**：很多项目要求提交Word格式的文档
- **便于编辑**：Word格式便于最终的修改和调整
- **便于分享**：Word格式兼容性好，便于分享和协作
- **便于打印**：Word格式便于打印和装订

### 转换方法

#### 方法1：转换单个文档
```bash
python3 scripts/markdown_to_word.py -i references/概要设计模板.md -o output.docx -t "项目概要设计"
```

#### 方法2：批量转换所有生成的文档
```bash
python3 scripts/markdown_to_word.py -d generated_docs/ -o word_output/
```

#### 方法3：转换参考模板
```bash
python3 scripts/markdown_to_word.py -d references/ -o word_templates/
```

### 转换参数说明
- `-i` 或 `--input`: 输入的markdown文件路径
- `-o` 或 `--output`: 输出的Word文件路径或目录
- `-d` 或 `--directory`: 输入目录路径（批量转换）
- `-t` 或 `--title`: 文档标题（单个文件转换时使用）
- `-p` 或 `--pattern`: 文件匹配模式（默认为`*.md`）

### 转换后的文件位置
转换后的Word文件默认保存在：
- 单个文件：指定的输出路径
- 批量转换：`word_output/` 目录下
- 参考模板：`word_templates/` 目录下

### Word文档特点
转换后的Word文档具有以下特点：
- ✅ **格式规范**：自动应用标题样式、段落格式
- ✅ **表格支持**：markdown表格转换为Word表格
- ✅ **代码块支持**：代码块以等宽字体显示
- ✅ **中文支持**：自动使用中文字体（黑体、宋体）
- ✅ **样式统一**：统一的标题层级和段落样式
